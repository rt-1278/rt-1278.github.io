経歴の詳細
新卒で入社したカー用品量販店で働くも、平均4~5時間の残業を繰り返したことで体調を崩して退職。
退職後は、IT技術を用いて社会貢献できる人材になりたいと思い、ITエンジニアになることを決意。
そして、半年間はアルバイトをしつつ、プログラミングスクールに通うための学費を貯蓄。
翌年度から約1年間、プログラミングスクールでJava言語の学習を行い、卒業間近にOCJPのSilverを取得し卒業。
翌年度から、ベンチャー系SIer企業に入社。

ベンチャー系SIerでの活動
  • 大手情報サービス企業様のお仕事マッチングサービス（クライアント向けWebアプリ、APIサーバ、モバイルアプリの開発改修）
  • 大手化粧品メーカー様のECサイトの改修とサーバ保守
  • システムエンジニア向けの新入社員研修の講師
などに携わってきました。

プライベートの活動
プライベートで作成したアプリケーションはAndroidアプリを２つとAPIサーバーを１つ。
最近は主にAndroidアプリ開発やAPIサーバーの構築などを行い
要件定義からデプロイそしてサーバ保守で必要となるあらゆる技術を
習得することを心がけて日々取り組んでいます。

クラウド


クラウド 着手内容
Amazon Web Services（AWS） 経験としては自分で作成したAPIサーバーをリリースする際ドメイン取得からデプロイまで全てAWSで行った経験
Google Cloud Platform（GCP） スマホアプリの外部APIとして利用
Firebase 業務でもプライベートでも利用。業務ではCloud Messaging, A/Bテスト, Crashlyticsなど様々な機能を利用。個人利用では主にCrashlyticsを利用して不具合対応で活用
  • 個人用のAPIサーバーをリリースする際にAWSを利用して、サーバーを構築しました。業務でサーバ保守の作業が FIXME るのですがデータセンター事業者のサーバーにデプロイしている状況で私個人はデプロイ作業や設定周りに携わっていないです。GCPとFirebaseはAPI利用とクラッシュ対応をメインに利用しています。
